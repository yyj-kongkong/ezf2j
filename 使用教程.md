#### yml配置
```yml
# 媒体服务器配置
# 媒体服务器配置
ezf2j:
  media:
    enabled: true                    # 是否启用（默认 true）
    port: 8866                       # 监听端口（默认 8866）
    host: 0.0.0.0                    # 监听地址（默认 0.0.0.0）
    net-timeout: "15000000"          # 网络超时（微秒，默认 15 秒）
    read-or-write-timeout: "15000000" # 读写超时（微秒，默认 15 秒）
    auto-close: true                 # 无人观看自动关闭（默认 true）
    no-clients-duration: 60000       # 无人观看多久关闭（毫秒，默认 60 秒）
    enable-ffmpeg: false             # 是否启用 FFmpeg（默认 false=JavaCV）
    ffmpeg-path:                     # FFmpeg 路径（可选，默认自动查找）
    log-level: INFO                  # 日志级别

    # HLS 配置
    hls:
      port: 8866                     # HLS 监听端口
      host: localhost                # HLS 主机地址


```

#### 注入并使用
```java
 @Autowired
private SpringMediaEngine mediaEngine;

/**
 * 启动推流
 */
@PostMapping("/start")
public String startStream(@RequestParam String url) {
    boolean success = mediaEngine.startStream(url);
    return success ? "推流成功" : "推流失败";
}
```

#### 手动初始化
```java

 MediaEngine engine = new ManualMediaInitializer()
        .setConfig(new CustomizableMediaConfig()
                .setPort(8866)
                .setHost("0.0.0.0")
                .setAutoClose(true)
                .setNoClientsDuration(60000)
                .enableFFmpeg()  // 启用 FFmpeg
        )
        .initFFmpeg()           // 步骤 1：初始化 FFmpeg
        .createComponents()     // 步骤 2：创建组件
        .startServer()          // 步骤 3：启动服务器
        .complete();            // 步骤 4：完成

// 开始推流
        engine.startStream("rtsp://...");
// 5. 停止推流
// engine.stopStream(rtspUrl);
```

这样就完成了！启动 Spring Boot 后，媒体服务器会自动运行，可以直接调用 API 进行推流了。🎉

## 📊 核心 API 说明

### MediaEngine / SpringMediaEngine

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `startStream(url)` | 启动推流（最简单） | url: 流地址 | boolean: 是否成功 |
| `startStream(url, useFFmpeg)` | 启动推流（指定转码方式） | useFFmpeg: 是否用 FFmpeg | boolean |
| `startStream(url, useFFmpeg, autoClose, duration)` | 启动推流（完整配置） | autoClose: 自动关闭<br>duration: 关闭时长 | boolean |
| `stopStream(url)` | 停止推流 | url: 流地址 | void |
| `getWsPlayUrl(url, host, port)` | 获取 WebSocket 播放地址 | host/port: 可选 | String |
| `getHttpPlayUrl(url, host, port)` | 获取 HTTP-FLV 播放地址 | host/port: 可选 | String |
| `getAllActiveStreams()` | 获取所有活跃流 | 无 | String[] |
| `hasStream(url)` | 检查流是否存在 | url: 流地址 | boolean |
| `getActiveStreamCount()` | 获取活跃流数量 | 无 | int |

## 🎉 特性亮点

✅ **双模式支持** - Spring Boot 自动装配 / 非 Spring 手动初始化  
✅ **自动关流** - 无人观看自动关闭，节省资源  
✅ **多种转码方式** - JavaCV / FFmpeg 自由切换  
✅ **协议丰富** - 支持 RTSP、RTMP、HTTP-FLV、WebSocket-FLV、HLS  
✅ **易于集成** - 一行代码即可开启推流  
✅ **生产可用** - 自带监控、健康检查、日志记录
